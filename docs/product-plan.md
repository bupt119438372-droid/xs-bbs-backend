# 新生论坛产品计划书

## 1. 项目定位

- 产品名称：新生论坛
- 核心一句话：让“会吃灰的念头”被记住、被整理、被连接。
- 产品目标：围绕“念头发布 - 相似匹配 - 关系建立 - 每日启发”形成闭环，帮助用户认识自己、连接同频的人。
- 项目价值：既有社区属性，也有 LLM 赋能的内容理解和推荐属性，适合作为“社区 + 推荐 + 搜索/匹配 + AI”综合型秋招项目。

## 2. 目标用户

- 校园青年：情绪表达欲强，但不愿意高频发公开朋友圈。
- 初入职场用户：希望沉淀个人思考、寻找同频伙伴。
- 泛内容社区用户：偏好轻表达、弱社交、强共鸣。

## 3. 核心问题

- 念头分散在备忘录、微博、纸条里，长期无人整理。
- 用户想找到“懂自己的人”，但陌生人社交成本高。
- 内容平台强调热点，不强调个人长期价值和精神成长。

## 4. 核心卖点

1. 念头即内容单元，支持低门槛表达，不要求长文创作。
2. LLM 负责理解、归类、提炼、生成启发，不只是“聊天机器人”。
3. 相似念头驱动社交，从匿名共鸣逐渐走向实名连接。
4. 每日推送让沉淀的念头继续生长，形成长期陪伴感。

## 5. 功能设计

### 5.1 念头发布

- 用户可发布短文本“念头”。
- 发布时可设置“念度”：
  - 杂念：轻度、随手记录
  - 心念：有持续关注
  - 执念：高权重、强情绪和长期牵引
- 支持标签、情绪、是否匿名、是否允许被推荐。

### 5.2 相似念头发现

- 发布念头后，系统实时计算与其他用户历史念头的相似度。
- 若相似度达到单条阈值，用户可看到对方匿名念头摘要。
- 对方收到“有人和你产生了同频”的推送。

### 5.3 用户相似值与关系升级

- 用户之间维护“念头相似值”。
- 达到第一挡：匿名互见，可查看更多相似念头。
- 达到第二挡：实名互见，可跳转对方主页。
- 双向关注后：开放查看彼此全部公开念头。

### 5.4 每日念头启发

- LLM 每日基于用户近期念头、历史主题和关注关系生成启发内容。
- 内容来源包括：
  - 历史故事
  - 诗词金句
  - 名人采访
  - 书摘
  - 新闻人物表达
- 输出形式：
  - 一句话提醒
  - 一段解读
  - 一条行动建议

## 6. 相似值规则设计

### 6.1 念度权重

- 杂念：1.0
- 心念：1.5
- 执念：2.0

### 6.2 单条念头相似分

```text
单条相似分 = 70% * 文本语义相似度 + 30% * 念度匹配系数

其中：
念度匹配系数 = min(念度权重A, 念度权重B) / max(念度权重A, 念度权重B)
```

### 6.3 用户间总相似值

```text
用户相似值 = min(
  100,
  相似念头对数量 * 12
  + 相似念头的平均单条相似分 * 40
  + 相似念头的平均念度权重 * 8
)
```

### 6.4 关系挡位

- 0-29：仅内容留存，无匹配展示
- 30-59：匿名共鸣，可看到对方匿名摘要
- 60-100：实名解锁，可进入主页并发起关注

## 7. 用户核心流程

1. 注册登录
2. 发布第一条念头
3. 收到同频匹配反馈
4. 浏览匿名共鸣内容
5. 相似值累积后解锁实名关系
6. 双向关注后形成稳定社交链路
7. 每日收到 AI 启发和回访提醒

## 9. 前端技术选型

- Web 前端：Vue 3 + TypeScript + Vite + Pinia + Vue Router
- 组件库：Element Plus
- 样式方案：UnoCSS + CSS Variables
- 数据可视化：ECharts
- 通知与实时：SSE 优先，后续扩展 WebSocket

选择理由：

- Vue 3 生态成熟，开发效率高，适合快速搭建作品级项目。
- TypeScript + Pinia + Router 组合更利于体现工程化能力。
- Element Plus 适合作品快速稳定落地，后续可在视觉上二次包装。
